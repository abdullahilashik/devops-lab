# Production Deployment Guide: NestJS (REST API / GraphQL)

This guide provides a professional, step-by-step workflow for deploying a NestJS application on Ubuntu 24.04 LTS using PM2 and Nginx.

---

## 1. Server Setup

### System Update & Core Tools
```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y curl git build-essential
```

### Node.js 20+ Installation
```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
```

### PM2 Global Installation
PM2 is the industry standard for managing Node.js processes in production.
```bash
sudo npm install pm2 -g
```

---

## 2. Application Deployment

### Project Initialization
```bash
# Create app directory
sudo mkdir -p /var/www/nestjs-app
sudo chown $USER:$USER /var/www/nestjs-app
cd /var/www/nestjs-app

# Clone project
git clone <your-repo-url> .
```

### Build Process
NestJS is written in TypeScript and must be compiled to JavaScript for production.
```bash
# Install dependencies
npm install

# Build project
npm run build
```
**The `dist/` folder**: The build command generates a `dist` directory. This contains the compiled JavaScript files that the Node.js runtime executes. You should never run `npm run start:dev` in production.

### Environment Configuration
```bash
cp .env.example .env
nano .env
```
Ensure `NODE_ENV=production` is set.

---

## 3. Running NestJS in Production

### Start with PM2
Run the compiled entry point (`main.js`) located in the `dist` folder.
```bash
# Start the application
pm2 start dist/main.js --name "nestjs-api"

# Save process list to restart on server reboot
pm2 save
```

### Enable Auto-restart on Reboot
```bash
pm2 startup
# Copy and run the command generated by the output of the above line
```

### PM2 Monitoring Commands
- `pm2 status`: List all managed processes.
- `pm2 logs nestjs-api`: Tail real-time application logs.
- `pm2 restart nestjs-api`: Restart the app after a code update.
- `pm2 monit`: Dashboard for CPU and Memory usage.

## 4. Nginx Reverse Proxy Configuration

Nginx forwards external traffic from port 80/443 to the internal NestJS port (default 3000).

### Install Nginx
```bash
sudo apt install -y nginx
```

### Configure Server Block
```bash
sudo nano /etc/nginx/sites-available/nestjs-api
```
Paste the following configuration:
```nginx
server {
    listen 80;
    server_name api.your-domain.com;

    # Security Headers
    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-XSS-Protection "1; mode=block";
    add_header X-Content-Type-Options "nosniff";

    location / {
        proxy_pass http://localhost:3000; 
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### Enable Configuration
```bash
sudo ln -s /etc/nginx/sites-available/nestjs-api /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

## 5. Database Setup

### Option A: PostgreSQL (Recommended)
```bash
sudo apt install -y postgresql postgresql-contrib
sudo -u postgres psql
```
Inside PostgreSQL shell:
```sql
CREATE DATABASE nestjs_db;
CREATE USER nest_user WITH PASSWORD 'SecurePassword123!';
GRANT ALL PRIVILEGES ON DATABASE nestjs_db TO nest_user;
\q
```

### Option B: MySQL
```bash
sudo apt install -y mysql-server
sudo mysql
```
Inside MySQL shell:
```sql
CREATE DATABASE nestjs_db;
CREATE USER 'nest_user'@'localhost' IDENTIFIED BY 'SecurePassword123!';
GRANT ALL PRIVILEGES ON nestjs_db.* TO 'nest_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

### NestJS Connection String (.env)
Depending on your ORM (TypeORM, Prisma, Sequelize), your `.env` should look like:
```env
# PostgreSQL Example
DATABASE_URL="postgresql://nest_user:SecurePassword123!@localhost:5432/nestjs_db"

# MySQL Example
DB_HOST=localhost
DB_PORT=3306
DB_USERNAME=nest_user
DB_PASSWORD=SecurePassword123!
DB_DATABASE=nestjs_db
```

## 7. Domain & Access Setup

### Domain Setup (Recommended)
1. **DNS A Record**: Point `api.your-domain.com` $\rightarrow$ `Server Public IP`.
2. **Nginx**: Ensure `server_name` in Nginx config matches this domain.

### IP-Based Access
If you have no domain, you can access the app via `http://<server-ip>:3000`. 
*Note: You must open port 3000 in your Cloud/VPS firewall (e.g., AWS Security Groups).*

## 8. SSL (HTTPS Setup)

### Install Certbot
```bash
sudo apt install -y certbot python3-certbot-nginx
```

### Generate Certificate
```bash
sudo certbot --nginx -d api.your-domain.com
```
Certbot will automatically modify the Nginx config to redirect HTTP $\rightarrow$ HTTPS.

### Auto-Renewal
```bash
sudo certbot renew --dry-run
```

## 9. Performance Optimization

### Nginx Gzip
Enable gzip in `/etc/nginx/nginx.conf` or your server block:
```nginx
gzip on;
gzip_types text/plain application/json application/javascript text/css;
```

### PM2 Cluster Mode
For multi-core servers, run multiple instances of your app to handle more traffic:
```bash
pm2 start dist/main.js --name "nestjs-api" -i max
```

### Memory Tuning
If the app crashes under load, increase Node.js memory limit in PM2:
```bash
pm2 start dist/main.js --name "nestjs-api" --node-args="--max-old-space-size=2048"
```

## 11. Common Issues & Fixes

| Issue | Cause | Resolution |
|---|---|---|
| **502 Bad Gateway** | NestJS app not running or port mismatch | Check `pm2 status` $\rightarrow$ Verify port in `.env` vs Nginx `proxy_pass` |
| **App not starting** | `dist` folder missing | Run `npm run build` again |
| **DB Connection Refused**| DB service stopped or wrong credentials | `sudo systemctl status postgresql` $\rightarrow$ Check `.env` |
| **Port 3000 occupied** | Ghost process running | `sudo lsof -i :3000` $\rightarrow$ `kill -9 <PID>` |
| **Module not found** | `npm install` skipped/failed | Run `npm install` and rebuild |

## 12. Optional Advanced Enhancements

### Low RAM Swap Setup
For 1GB RAM servers, add 2GB swap to prevent build crashes:
```bash
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

### Scaling Path
- **Horizontal Scaling**: Deploy multiple EC2 instances $\rightarrow$ Put an AWS Application Load Balancer (ALB) in front $\rightarrow$ Move DB to AWS RDS.
- **CI/CD**: Use GitHub Actions to SSH into the server, `git pull`, `npm install`, `npm run build`, and `pm2 reload all`.
