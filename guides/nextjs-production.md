# Production Deployment Guide: Next.js (SSR + API Routes)

This guide provides a systematic approach to deploying a Next.js application on Ubuntu 24.04 LTS using PM2 and Nginx.

---

## 1. Server Setup

### System Update & Core Tools
```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y curl git build-essential
```

### Node.js 20+ Installation
Use the NodeSource repository to ensure the correct LTS version.
```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
```
Verify installations:
```bash
node -v # Should be v20.x.x
npm -v
```

---

## 2. Application Deployment

### Project Initialization
```bash
# Create app directory
sudo mkdir -p /var/www/next-app
sudo chown $USER:$USER /var/www/next-app
cd /var/www/next-app

# Clone project
git clone <your-repo-url> .
```

### Dependency Installation & Build
```bash
# Install dependencies
npm install

# Configure environment variables
cp .env.example .env.production
nano .env.production

# Build the application
npm run build
```
**The `.next` folder**: The build process creates a `.next` directory containing the optimized production build (compiled JS, CSS, and server-side chunks). This is what the Node.js server executes.

---

## 3. Process Management (PM2)

PM2 ensures your Next.js server runs in the background and restarts automatically on crashes or server reboots.

### Install & Start
```bash
# Install PM2 globally
sudo npm install pm2 -g

# Start Next.js app
# 'npm start' runs 'next start' which launches the production server on port 3000
pm2 start npm --name "next-app" -- start

# Save current process list for reboot
pm2 save
```

### Enable Auto-start on Reboot
```bash
pm2 startup
# Copy and run the command generated by the output of the above line
```

### Essential PM2 Commands
- `pm2 status`: View running processes
- `pm2 logs next-app`: Tail application logs
- `pm2 restart next-app`: Restart application
- `pm2 stop next-app`: Stop application
- `pm2 monit`: Real-time CPU/Memory monitor

## 4. Nginx Reverse Proxy Configuration

Nginx acts as the entry point, handling SSL and forwarding requests to the Next.js process running on port 3000.

### Install Nginx
```bash
sudo apt install -y nginx
```

### Configure Server Block
```bash
sudo nano /etc/nginx/sites-available/next-app
```
Paste the following configuration:
```nginx
server {
    listen 80;
    server_name your-domain.com www.your-domain.com;

    # Gzip Compression
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # Cache static assets from .next/static
    location /_next/static {
        proxy_pass http://localhost:3000;
        proxy_cache_bypass $http_upgrade;
        expires 365d;
        add_header Cache-Control "public, max-age=31536000, immutable";
    }
}
```

### Enable Configuration
```bash
sudo ln -s /etc/nginx/sites-available/next-app /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

## 5. Domain & IP Access

### Domain Setup (Recommended)
1. **DNS A Record**: In your domain provider (Cloudflare, Route53, etc.), create an `A` record pointing `@` and `www` to your server's Public IP.
2. **Nginx**: Ensure `server_name` matches your domain.

### IP-Based Access (Alternative)
If you do not have a domain, you can access the app directly via:
`http://<server-ip>:3000`
*(Note: This requires opening port 3000 in your firewall/security group)*.

## 6. SSL (HTTPS)

### Certbot Installation
```bash
sudo apt install -y certbot python3-certbot-nginx
```

### Generate Certificate
```bash
sudo certbot --nginx -d your-domain.com -d www.your-domain.com
```
Certbot will automatically update your Nginx config to redirect HTTP to HTTPS.

### Auto-Renewal
Certbot adds a timer to systemd automatically. Verify it:
```bash
sudo certbot renew --dry-run
```

## 7. Performance Optimization

### Memory Management
Next.js builds can be memory-intensive. For low-RAM servers, implement a swap file:
```bash
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

### Node Memory Limit
If the app crashes due to "Out of Memory", increase the Node.js heap limit in PM2:
```bash
pm2 start npm --name "next-app" -- node --max-old-space-size=2048 node_modules/.bin/next start
```

## 8. Production Stability

### Restart Strategy
When deploying updates:
1. `git pull`
2. `npm install`
3. `npm run build`
4. `pm2 restart next-app`

### Log Monitoring
Use PM2 logs to debug runtime errors:
```bash
pm2 logs next-app --lines 100
```

## 9. Common Issues & Fixes

| Issue | Diagnosis | Resolution |
|---|---|---|
| **502 Bad Gateway** | Next.js app is not running on port 3000 | `pm2 status` $\rightarrow$ `pm2 start next-app` |
| **Build Failure** | Insufficient RAM during `npm run build` | Setup Swap memory (see Section 7) |
| **Port 3000 occupied**| Another process is using the port | `sudo lsof -i :3000` $\rightarrow$ `kill -9 <pid>` |
| **Env Vars not loading**| Using `.env.local` instead of `.env.production` | Ensure `.env.production` exists and is loaded |
| **Permission Denied** | App cannot write to `.next` or `public` | `sudo chown -R $USER:$USER /var/www/next-app` |

## 10. Optional Advanced Enhancements

### Multi-App Hosting
To host multiple Next.js apps on one server:
1. **Assign Different Ports**: App 1 on port 3000, App 2 on port 3001.
   - Change port in `package.json`: `"start": "next start -p 3001"`
2. **Nginx Subdomains**: Create separate server blocks for `app1.domain.com` (proxy to 3000) and `app2.domain.com` (proxy to 3001).
